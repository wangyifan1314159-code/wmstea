# 矿井掘进远程操控系统 - 技术架构与协议详解

## 一、系统总体架构

远程操控系统采用**四层架构**设计，实现井下设备的远程控制、实时监控和智能决策：

1. **井下感知与控制层** - Underground Sensing & Control Layer
2. **网络传输层** - Network Transmission Layer
3. **地面集控中心** - Ground Control Center
4. **安全生态系统** - Safety Ecosystem

## 二、核心技术栈

### 2.1 定位与导航技术

#### 光纤惯导系统 (FOG-INS)
- **技术原理**: 光纤陀螺仪 + 加速度计组合惯性导航
- **定位精度**: ≤5mm/100m
- **输出频率**: 100Hz
- **数据接口**: RS-422/RS-485
- **传输协议**: 自定义二进制协议 / Modbus RTU
- **应用场景**: 掘进机实时姿态感知、累积位移计算

#### 激光定位系统
- **技术方案**: 激光基站 + 反射靶标
- **绝对定位精度**: ±2mm
- **测量范围**: 0-200m
- **更新频率**: 10Hz
- **通信协议**: TCP/IP over Ethernet
- **数据格式**: JSON / XML
- **作用**: 修正惯导累积误差，提供绝对位置基准

#### 视觉SLAM
- **算法框架**: ORB-SLAM3 / LSD-SLAM
- **传感器**: 双目/RGB-D摄像头
- **点云密度**: 10,000-50,000点/帧
- **建图精度**: 相对精度 1-2%
- **计算平台**: NVIDIA Jetson AGX Xavier
- **传输带宽**: 需50-100Mbps
- **应用**: 迎头环境三维重建、障碍物检测

### 2.2 通信传输协议

#### TRUNB技术 + 5G专网融合
```
TRUNB技术核心特性:
- 国产自研可控: 自主知识产权，核心芯片与协议栈完全国产化，满足信创要求
- 高宽带: 峰值吞吐量≥200Mbps（井下复杂电磁环境适配）
- 低时延: 端到端控制指令延迟≤10ms，远优于传统工业无线方案
- 远程遥控专项优化: 专为工业远控场景设计，抗干扰能力强，信道资源独立调度

TRUNB + 5G协同工作模式:
- TRUNB通道: 承载安全关键控制指令（硬实时，低时延优先）
- 5G eMBB切片: 承载高清视频、SLAM点云数据（高宽带优先）
- 双通道互备: 任一链路故障自动切换，切换时间<50ms

远程遥控+辅助纠偏高精度实现:
- 远程操控台→TRUNB通道→远控执行控制器，操控指令闭环≤20ms
- 纠偏触发条件: 惯导/激光双传感器融合检测偏差>5mm
- 辅助纠偏算法: 卡尔曼滤波融合位姿估计 → PID轨迹修正 → 液压纠偏机构执行
- 纠偏精度: 横向偏差≤±10mm，方向偏差≤±0.1°
```

#### 5G专网
```
核心参数:
- 频段: n78 (3.5GHz) / n79 (4.9GHz)
- 带宽: 100MHz
- 上行峰值速率: ≥100Mbps
- 下行峰值速率: ≥500Mbps
- 端到端延迟: ≤20ms (uRLLC)
- 抖动: ≤5ms
- 丢包率: ≤10⁻⁵

网络切片配置:
1. 控制指令切片 (uRLLC)
   - 带宽预留: 10Mbps
   - 延迟保证: ≤10ms
   - 优先级: 最高

2. 视频监控切片 (eMBB)
   - 带宽预留: 50Mbps
   - 编码格式: H.265/HEVC
   - 分辨率: 1080P @30fps
   - 延迟要求: ≤150ms

3. 传感器数据切片 (mMTC)
   - 带宽预留: 20Mbps
   - 连接密度: ≥100设备/基站
   - 延迟要求: ≤100ms
```

#### 工业以太网
```
拓扑结构: 双冗余环网 (Ring Topology)
协议栈:
├─ 物理层: IEEE 802.3 (千兆/万兆)
├─ 数据链路层: RSTP (快速生成树) / ERPS (以太网环网保护)
├─ 网络层: IPv4 / IPv6
└─ 应用层: PROFINET / EtherCAT / OPC UA

PROFINET实时通信:
- RT (Real-Time): 循环周期 10-100ms
- IRT (Isochronous Real-Time): 循环周期 ≤1ms
- 抖动: ≤1μs
- 时钟同步: IEEE 1588 PTP

EtherCAT总线:
- 循环时间: 100μs - 1ms
- 拓扑: 线型/树型/环型
- 从站数量: ≤65535
- 应用: 截割电机、液压阀组等高速控制
```

#### OPC UA协议
```
架构特性:
- 传输层: TCP/IP (opc.tcp://) 或 HTTPS
- 安全模式:
  * None (仅测试环境)
  * Sign (消息签名)
  * Sign & Encrypt (签名+AES-256加密)

- 认证方式:
  * 匿名
  * 用户名/密码
  * X.509证书

数据模型:
- 对象 (Objects): 掘进机、截割系统、液压系统
- 变量 (Variables): 油温、油压、电流、转速
- 方法 (Methods): 启动截割、停止截割、紧急制动
- 事件 (Events): 故障告警、维护提醒

服务集:
1. 发现服务 (Discovery): 自动发现OPC UA服务器
2. 会话服务 (Session): 建立安全连接
3. 订阅服务 (Subscription): 实时数据推送
4. 历史数据访问 (Historical Access): 历史曲线查询
```

## 三、远程操控逻辑实现

### 3.1 控制指令流

```
操作台 → 集控平台 → 加密模块 → 5G/工业网 → 边缘网关 → 联锁验证 → 执行控制器 → 设备动作
   ↑                                                                              ↓
   └───────────────────────── 状态反馈 (100ms周期) ──────────────────────────────┘

详细步骤:
1. 操作员通过六自由度手柄发出控制指令
   - 前进/后退/左转/右转/抬升/下降
   - 截割启动/停止/调速
   - 喷雾开启/关闭

2. 地面集控平台接收指令 (10ms内)
   - 指令格式验证 (JSON Schema)
   - 操作权限检查 (RBAC)
   - 记录审计日志

3. 加密模块处理 (5ms内)
   - AES-256-GCM加密
   - 添加时间戳防重放
   - 数字签名 (RSA-2048)

4. 网络传输 (10-20ms)
   - 5G uRLLC切片 / 工业以太网
   - QoS优先级调度
   - 丢包重传机制

5. 井下边缘网关接收 (5ms内)
   - 解密验证签名
   - 协议转换 (JSON → Modbus/PROFINET)
   - 缓存队列管理

6. 联锁安全验证 (10ms内)
   - 检查环境参数 (瓦斯、顶板)
   - 检查人员位置 (电子围栏)
   - 检查设备状态 (故障闭锁)
   - 验证通过 → 执行; 不通过 → 拒绝+告警

7. 执行控制器动作 (20-50ms)
   - PLC/继电器输出
   - 液压阀切换
   - 电机变频调速
   - 实时状态采样

8. 状态反馈上传 (周期100ms)
   - 设备位姿 (惯导)
   - 液压压力、油温
   - 电机电流、转速
   - 视频画面同步
```

### 3.2 安全联锁矩阵

| 触发条件 | 联锁动作 | 响应时间 | 优先级 |
|---------|---------|---------|--------|
| 瓦斯浓度 ≥1.0% | 闭锁截割系统，禁止启动 | ≤100ms | P1 |
| 瓦斯浓度 ≥1.5% | 切断全部非本安电气设备电源 | ≤50ms | P0 |
| 人员进入危险区 (距迎头<5m) | 掘进机全部动作停止并闭锁 | ≤200ms | P1 |
| 顶板离层 >30mm | 禁止掘进机移机，允许截割 | ≤500ms | P2 |
| 顶板离层 >50mm | 全部设备停机，启动支护应急 | ≤300ms | P1 |
| 设备温度 >85℃ | 自动降功率至50%，持续监测 | ≤1s | P3 |
| 设备温度 >95℃ | 紧急停机，闭锁设备 | ≤500ms | P2 |
| 液压油温 >70℃ | 开启强制冷却，降低工作压力 | ≤2s | P3 |
| 液压油压 <15MPa | 闭锁截割动作，告警检修 | ≤1s | P2 |
| 电机电流 >额定值120% | 自动卸载，持续3s未恢复则停机 | ≤500ms | P2 |
| 粉尘浓度 >200mg/m³ | 强制启动喷雾，降低截割速度 | ≤1s | P3 |
| 红色灾害预警 (AI判定) | 全工作面设备紧急停机+人员撤离 | ≤1s | P0 |
| 通信中断 >5s | 井下设备自动保持当前状态，禁止新动作 | 实时检测 | P1 |
| 急停按钮按下 | 切断主电源，所有设备立即制动 | ≤50ms | P0 |

**联锁实现方式:**
- **硬接线联锁** (P0/P1级): 传感器 → 安全继电器 → 接触器线圈，旁路PLC/网络
- **PLC逻辑联锁** (P2/P3级): 传感器 → PLC输入 → 逻辑运算 → 输出控制
- **软件联锁** (补充验证): 边缘网关 + 地面平台双重校验

### 3.3 操作权限分级

```
Level 1: 观察员 (Observer)
- 只读权限: 查看实时视频、数据
- 禁止操作: 无法发送任何控制指令
- 认证方式: 账号密码

Level 2: 操作员 (Operator)
- 标准操作: 常规掘进、截割、移机
- 禁止操作: 参数修改、联锁解除、应急复位
- 认证方式: 人脸识别 + 虹膜扫描
- 授权时间窗口: 班次内有效 (8小时)

Level 3: 工程师 (Engineer)
- 高级操作: 参数调整、故障诊断、手动单步调试
- 禁止操作: 联锁强制解除 (需审批)
- 认证方式: 双因子认证 (硬件Key + 生物特征)
- 操作审计: 全程录像 + 区块链存证

Level 4: 管理员 (Administrator)
- 最高权限: 系统配置、联锁参数修改、应急解锁
- 操作约束: 需两人同时在场确认 (双人双锁)
- 认证方式: 三因子认证 + 短信OTP
- 审批流程: 电子审批单 + 上级主管远程授权
```

## 五、AI算法应用

### 5.1 灾害预警模型

#### 瓦斯突出预警
```python
模型架构: LSTM + Attention

输入特征 (时间序列, 窗口=60分钟):
- 瓦斯浓度 (CH4, CO2)
- 瓦斯涌出量
- 气压差
- 掘进速度
- 煤层地质参数 (倾角、硬度)

模型结构:
Input(60, 8)
  → LSTM(128, return_sequences=True)
  → Attention Layer
  → LSTM(64)
  → Dropout(0.3)
  → Dense(32, activation='relu')
  → Dense(4, activation='softmax')  # 4级预警

输出:
- Class 0: 正常 (绿色)
- Class 1: 关注 (蓝色)
- Class 2: 预警 (黄色)
- Class 3: 危险 (红色)

训练数据: 300个工作面历史数据, 50万条样本
准确率: 92.5%
```

### 6.2 逃生路线规划算法

```python
算法: Dijkstra最短路径 + 动态障碍物规避

图模型构建:
- 节点: 巷道交叉口、避险硐室、安全出口、人员当前位置
- 边: 巷道路段
- 权重: 距离 + 实时危险度评分

动态权重计算:
weight = base_distance × (1 + danger_factor)

danger_factor =
  + 0.5 × (gas_concentration / threshold)
  + 0.3 × (smoke_density / max_density)
  + 0.2 × (temperature / critical_temp)
  + 1.0 × is_blocked  # 巷道堵塞则权重×2

实时更新频率: 每5秒重新计算
推送方式:
- 人员定位卡震动 + LED箭头指示
- 应急广播语音引导
- 地面大屏实时显示所有人员逃生轨迹

优化目标:
1. 优先选择最短安全路径
2. 均衡分配避险空间 (避免拥挤)
3. 考虑老弱病残人员的移动能力
```

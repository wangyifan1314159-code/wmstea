① 感知层 — 数据源头
将井下物理环境、设备状态和人员行为转化为数字信号
Modbus RTU / RS485

核心作用

将井下的物理环境、设备状态和人员行为转化为可传输的数字信号（0和1），是整个系统的数据源头。

🔌 通信协议

Modbus RTU
工业传感器最通用协议，负责将数据汇集到边缘网关
RS485
物理层标准，支持多设备总线通信，抗干扰能力强

⚡ 数据流向

传感器 → RS485总线 → Modbus RTU → 边缘网关

☁️
瓦斯传感器
实时监测CH₄浓度，超阈值自动报警
💨
粉尘传感器
监测悬浮粉尘浓度，防范爆炸风险
📡
顶板激光雷达
毫米级扫描顶板形变，监测坍塌风险
📳
振动传感器
监测掘进机轴承振动，诊断设备健康
🌡️
温度传感器
监测电机及减速箱温度，预防过热故障
🏷️
UWB定位标签
厘米级人员实时定位，越界自动预警
② 网络层 — 传输通道
构建"有线+无线"高可靠网络，把数据从井下搬运到地面
TCP/IP + MQTT

核心作用

解决井下环境复杂、距离远的问题，构建一张"有线+无线"高可靠网络，负责把数据从井下搬运到地面。

🔌 通信协议

TCP/IP
网络基础，确保数据不丢包、可靠传输
MQTT
轻量级、抗弱网，适合海量传感器数据上传

⚡ 数据流向

边缘网关 → 5G/环网 → MQTT Broker → 平台层

📶
矿用5G基站
移动设备接入 & 高清视频传输，低延迟
🔀
工业环网交换机
固定设备有线冗余传输，双环网防断链
🔌
边缘计算网关
核心翻译官：Modbus → MQTT 协议转换
📹
高清摄像头
井下视频监控，AI分析违规行为
🔋
UWB基站
接收定位标签信号，计算坐标上传
🌐
光纤骨干网
井下到地面高速通道，千兆传输保障
③ 平台层 — 数据中枢
充当系统"大脑"，接收、存储、清洗、分析数据，提供标准接口
MQTT + HTTP API

核心作用

充当系统的"大脑"，负责接收海量数据、存储数据，并通过算法进行清洗和分析，为上层应用提供标准接口。

🔌 通信协议

MQTT
继续接收网关上来的传感器数据流
HTTP API
对外提供RESTful接口，供Web端和App调用
